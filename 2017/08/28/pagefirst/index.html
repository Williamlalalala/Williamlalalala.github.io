简单的hexo和GitHub搭建自己的博客

By Chenhw

Published 2017-08-28

Contents

hhhhhh，今天终于打好了博客，真心累，虽然我还不知道具体其他的改法，不急，慢慢学吧，那我就先按照打完博客的套路（233333）讲一下怎么做一个自己的博客吧
我是使用的hexo和GitHub搭建的

第一步先到网络上下载node.js 和git（使用GitHub必须链接在此）

其实下载没什么说的，没有什么限制之类的，基本上是按照它的默认来，建议node.js放到D盘中，先建立一个blog的文件夹，再在里面建一个nodejs的文件夹，然后把node.js放在那里面，以后那就是你博客的根据地了。
下载完git之后打开git.exe可以先放在那里（以防找不到了）

然后就是把环境变量改一下具体如下


进入计算机的系统属性，先点更改设置再点环境变量


然后就会看到左边的窗口弹出，找到PATH然后添加你的node.js的路径

紧接着按win的键+R在其中输入cmd打开命令行

先进入你的blog文件夹再进入nodejs的文件夹

C:\Users\chenhw>D:
D:\>cd blog
D:\blog>cd nodejs
安装hexo
D:\blog\nodejs>npm install -g hexo
紧接着创建一个文件夹
D:\blog\nodejs>mkdir blog
然后进去
D:\blog\nodejs>cd blog
初始化hexo（这一步我用了200多秒）
D:\blog\nodejs\blog>hexo init 
紧接着安装依赖包
D:\blog\nodejs\blog>npm install
再输入npm install hexo-deployer-git --save

初始的工作我们就进行完了，然后我们现在本地进行部署

命令行输入
hexo generate（hexo g也可以）
hexo server（hexo s也可以）这里是本地部署，部署完别把cdm命令行关掉

紧接着我们就要把网页部署到GitHub上，我们去GitHub的官网（GitHub那个猫的标超级萌有木有）申请一个账号，


新建一个仓库，然后写入名字


紧接着我们create完之后回到主页找到setting

向下翻翻到GitHub pages的地方选择主题

然后紧接着我们再在GitHub里把ssh设置好，这里现在GitHub中进行操作，具体操作在这

紧接着我们打开没有关掉的git的命令行进行操作就可以了


这边创建ssh的部分就算完成了

接下来要配置Deployment
在_config.yml文件中，找到Deployment，然后按照如下修改（因为yml文件对于格式的要求特别高，所以要在deploy下面的每个属性前加两个空格，冒号后加一个空格）：

deploy:
type: git
repo: git@github.com:Williamlalalala/Williamlalalala.github.io.git
branch: master

我们再回到cmd命令行，输入

hexo g
hexo d

就可以部署成功了
